# 7. Requisitos do Sistema

## 7.1 Requisitos Funcionais (RF) 

RF01 – O sistema deve permitir o cadastro de alunos com dados pessoais (nome, telefone, faixa, etc.).  

RF02 – O sistema deve permitir editar e excluir dados dos alunos.  

RF03 – O sistema deve permitir o cadastro de professores.  

RF04 – O sistema deve permitir registrar a presença dos alunos em cada aula.  

RF05 – O sistema deve permitir consultar o histórico de frequência de cada aluno.  

RF06 – O sistema deve gerar relatórios de frequência por período.  

RF07 – O sistema deve permitir registrar a evolução técnica dos alunos.  

RF08 – O sistema deve permitir visualizar o histórico de desempenho individual.  

RF09 – O sistema deve permitir identificar dificuldades específicas dos alunos.  

RF10 – O sistema deve permitir registrar o pagamento das mensalidades.  

RF11 – O sistema deve permitir consultar a situação financeira do aluno (adimplente/inadimplente).  

RF12 – O sistema deve gerar relatórios financeiros básicos.  

RF13 – O sistema deve gerar relatórios integrados de frequência, desempenho e situação financeira.  

RF14 – O sistema deve permitir visualizar dados consolidados para apoio à tomada de decisão.  

 

## 7.2 Requisitos Não Funcionais (RNF) 

RNF01 – O sistema deve possuir interface simples e intuitiva.  

RNF02 – O sistema deve ser fácil de usar por usuários sem conhecimento técnico.  

RNF03 – O sistema deve responder às ações do usuário em até 2 segundos.  

RNF04 – O sistema deve suportar múltiplos acessos simultâneos sem perda de desempenho.  

RNF05 – O sistema deve exigir autenticação (login e senha).  

RNF06 – O sistema deve garantir a proteção dos dados dos alunos.  

RNF07 – O sistema deve restringir acessos conforme o tipo de usuário (ex: administrador, professor).  

RNF08 – O sistema deve estar disponível durante o horário de funcionamento da academia.  

RNF09 – O sistema deve ser estruturado de forma modular para facilitar manutenção e evolução.  

RNF10 – O sistema deve ser acessível via navegador web.
